## 2026-03-04 18:05 UTC

### Critical Discovery: Edge Config is Read-Only at Runtime! 🚨

**Problem:**
- Edge Config v1 API does NOT have `set()` function at runtime
- Can only READ values via `get()` and `getAll()`
- Can only WRITE via Vercel API or CLI (requires API token)
- Result: Cannot update agent status at runtime!

**Solution: Switched to Vercel Postgres**

**Why Postgres?**
- ✅ Full CRUD operations (CREATE, READ, UPDATE, DELETE)
- ✅ Native to Vercel
- ✅ Fully persistent (survives deployments)
- ✅ Free tier available (60 hours compute/month, 512 MB storage)
- ✅ Unlimited reads/writes
- ✅ SQL queries for filtering/sorting

## Database Schema

### agents_status Table
```sql
CREATE TABLE agents_status (
  agent_id VARCHAR(255) PRIMARY KEY,
  status VARCHAR(50),
  current_task TEXT,
  todos JSONB,
  last_updated TIMESTAMP WITH TIME ZONE DEFAULT NOW()
)
```

### agent_messages Table
```sql
CREATE TABLE agent_messages (
  id VARCHAR(255) PRIMARY KEY,
  from_agent VARCHAR(255),
  to_agent VARCHAR(255),
  message TEXT,
  type VARCHAR(50),
  read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
)
```

## Why Edge Config Failed

**Original Approach:**
```typescript
import { get, set } from '@vercel/edge-config'

const config = await getAll()
config[key] = value  // This doesn't work!
```

**Problem:**
- `getAll()` returns a copy of the config
- Modifying the copy doesn't update the actual config
- No `set()` function in v1 API

**Correct Approach (for Postgres):**
```typescript
import { sql } from '@vercel/postgres'

await sql`
  UPDATE agents_status
  SET status = ${status}
  WHERE agent_id = ${agentId}
`
```
